# P9748

## 题意

n 个苹果排成一列，每天从最左侧开始，每隔 2 个苹果拿 1 个（即拿走第 1, 4, 7, ... 个），剩下的重新排列。求：

1. 多少天拿完所有苹果
2. 编号为 n 的苹果在第几天被拿走

## 核心观察

每个苹果的位置由 **它前面还有多少个苹果没被拿** 决定，只需跟踪编号 n 的苹果的当前位置。

### 每天拿走的苹果数

当前有 m 个苹果时，拿走的位置为 1, 4, 7, ...（每 3 个拿第 1 个）：

- 拿走数量 = $\lceil m / 3 \rceil$（C++ 写法 `(m + 2) / 3`）
- 剩余数量 = $m - \lceil m/3\rceil = \lfloor 2m/3 \rfloor$

### 追踪编号 n 的苹果（位置法）

设当前剩 m 个苹果，编号 n 的苹果位于其中的第 p 个位置（1-indexed）。

**如果 $p \equiv 1 \pmod 3$（即 `p % 3 == 1`）** → 今天被拿走，记录天数。

**否则** → 它前面被拿走的苹果数 = $\lceil (p-1)/3\rceil = (p+1)/3$（整数除法），新位置：
$$p' = p - \left\lceil\frac{p-1}{3}\right\rceil = p - \frac{p+1}{3}$$

### 完整算法

```cpp
int days = 0, ans_n_day = 0;
int pos = n;          // 编号 n 当前在第 pos 个位置

while (n > 0) {
    days++;
    if (pos % 3 == 1 && !ans_n_day)
        ans_n_day = days;
    if (pos % 3 != 1)
        pos -= (pos + 1) / 3;   // 更新位置
    n -= (n + 2) / 3;           // 拿掉 ceil(n/3) 个
}
```

## 复杂度

每轮 n 至少减少为原来的 $2/3$，至多 $\log_{3/2} 10^9 \approx 52$ 轮，$O(\log n)$。

## 数据范围

$n \leq 10^9$，必须用数学/模拟法，不可 O(n) 模拟。
