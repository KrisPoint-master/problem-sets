## 题目解析

1. 第一步: 转换,
    - 离开一个数字的时候消耗1
    - 每个数字代表可以离开的次数
    - 0 表示不可以到达
2. `!!!从简单向复杂考虑: !!!本质是不同情况的下的思路,极限法--> 分情况讨论`
    - 非常容易想到: 直接按相邻的原则`-1,0,1`的原则走,遇到0就走`-2,2`,可以单方向的把能走的点都拿到
    - 只有一个数字
    - 两个数字
    - 三个数字
    - 4个数字
    - (写个暴力dfs)
    - 数字全部都是大于1的: 全部可以拿到: 答案就是所有数字的和
        - 为什么可以全部拿到(求和)? 思维转化: 交换验证: 所有的得到最大值的走法都可以转化成一种走法: 举出特殊的例子
        - `1 1 1 1 1` , `3 3 3 3 3`
    - 于是想到!!:  在起始位置:
        - 可以先向走到边界(1 0 1,0的旁边有个1,单向边),然后反向走到右边的边界
        - 也可以反过来

    - 显然想到如果有0怎么办(其他情况)
    - 1个0: `2 2 0 2 2`, `1 1 0 1 1` , `1 2 0 1 1`
        - 如果0的左右有一个0,那么就是一个单向的路径
    - 2个0 
        - 相邻`0 0`, 截断
        - 不相邻,就是只有一个0的情况

3. 如何解题呢(如何写代码?)

一旦越过单向边,就再也不能回头了,只能一直沿着前进的方向走

造几个数据,看看不同的情况


> !! 对于贪心来说,方法是固定的,因为: 交换验证
> 写一个暴力,真的可以帮你计算,思考
