
针对题目中 $n=500$ 的数据规模，我们完全可以使用 **二分答案 + 朴素动态规划 ($O(n^2)$)** 的方法来解决。

虽然这道题的正解（针对 $n=5\times 10^5$）需要用到“单调队列优化 DP”，但对于 $n=500$ 的部分分，朴素的 $O(n^2)$ 转移足以通过。

### 算法思路分析

这个问题可以分解为两部分：

1. **二分答案（确定花费 $g$）**：花费 $g$ 金币和“能否达到 $k$ 分”具有单调性。金币越多，跳跃范围越灵活，越容易达到目标分数。因此我们可以二分枚举 $g$。
2. **动态规划（Check 函数）**：对于固定的 $g$，判断机器人是否能获得至少 $k$ 分。

------

### DP 状态设计 (Check 函数)

假设当前花费为 $g$，则：

- 最小跳跃距离：$L = \max(1, d - g)$
- 最大跳跃距离：$R = d + g$

我们定义状态 $dp[i]$：

- **含义**：机器人跳到第 $i$ 个格子时，所能获得的最大分数。

- **初始状态**：$dp[0] = 0$（起点，设为第 0 格，分数为 0），其余初始化为负无穷大（$-\infty$），表示不可达。

- 转移方程：

  对于每个格子 $i$ (从 1 到 $n$)，我们遍历它前面的所有格子 $j$ (从 0 到 $i-1$)。如果 $j$ 到 $i$ 的距离 $dist = x_i - x_j$ 满足 $L \le dist \le R$，且 $dp[j]$ 是可达的，则尝试更新：

  

  $$dp[i] = \max(dp[i], dp[j] + s_i)$$

如果计算过程中发现任意 $dp[i] \ge k$，则返回 `true`（表示该花费 $g$ 可行）。

------

### 复杂度分析

- **二分复杂度**：$x_i$ 最大为 $10^9$，二分次数约为 $\log_2(10^9) \approx 30$ 次。

- **DP 复杂度**：对于每次 check，我们需要双重循环遍历。外层 $n$，内层 $n$，复杂度为 $O(n^2)$。

- **总复杂度**：$O(n^2 \log (\text{max\_dist}))$。

- 代入 $n=500$：$500^2 \times 30 = 250,000 \times 30 = 7.5 \times 10^6$。

  这远小于一般竞赛题 1 秒 (约 $10^8$ 次运算) 的限制，因此该解法对于 $n=500$ 非常稳健。
------
'''
#include <iostream>
#include <vector>
#include <algorithm>
#include <cstring>

using namespace std;

// 定义长整型，防止分数累加或坐标溢出 int
typedef long long ll;

const ll INF = 1e18; // 设置一个极小值代表不可达
const int MAXN = 5005; // 即使只针对 500，开大一点也无妨

int n;
ll d, k;
// 存储格子信息：位置 x 和 分数 s
struct Cell {
    ll x, s;
} cells[MAXN];

// dp 数组
ll dp[MAXN];

// 核心 Check 函数：判断花费 g 是否足够获得 k 分
bool check(ll g) {
    // 1. 计算当前的跳跃区间 [min_dist, max_dist]
    ll min_dist = max(1LL, d - g);
    ll max_dist = d + g;

    // 2. 初始化 DP 数组
    // dp[i] 表示跳到第 i 个格子时的最大分数
    for(int i = 1; i <= n; i++) dp[i] = -INF;
    dp[0] = 0; // 起点（虚拟的第0个点）分数为0

    // 3. 开始 DP (O(n^2) 暴力转移)
    for (int i = 1; i <= n; i++) {
        // 遍历之前的所有点 j
        for (int j = i - 1; j >= 0; j--) {
            ll dist = cells[i].x - cells[j].x;
            
            // 剪枝优化：如果距离已经超过最大跳跃距离，
            // 再往前找 j 距离只会更大，直接 break
            if (dist > max_dist) break;
            
            // 如果距离太近，不满足条件，continue 继续找更远的 j
            if (dist < min_dist) continue;

            // 如果 j 点是可达的，进行转移
            if (dp[j] != -INF) {
                dp[i] = max(dp[i], dp[j] + cells[i].s);
            }
        }
        
        // 优化：如果到了某个点已经满足分数要求，直接返回 true
        if (dp[i] >= k) return true;
    }

    return false;
}

int main() {
    // 优化 I/O
    ios::sync_with_stdio(false);
    cin.tie(0);

    // 输入
    cin >> n >> d >> k;
    
    // 初始化起点：位置0，分数0
    cells[0] = {0, 0};
    
    // 读入 n 个格子
    for (int i = 1; i <= n; i++) {
        cin >> cells[i].x >> cells[i].s;
    }

    // 二分答案
    // 下界是 0，上界是坐标最大值（稍微大一点保证覆盖）
    // 注意：如果有解，花费不会超过最大坐标值太多
    ll left = 0, right = 1000000005; 
    ll ans = -1;

    while (left <= right) {
        ll mid = (left + right) / 2;
        if (check(mid)) {
            ans = mid;      // 当前花费可行，尝试更小的花费
            right = mid - 1;
        } else {
            left = mid + 1; // 当前花费不可行，需要更多花费
        }
    }

    cout << ans << endl;

    return 0;
}
'''cpp
------
### 代码关键点说明

1. **数据类型**：分数 $k$ 和 $s_i$ 的累加可能很大，坐标 $x_i$ 也很大，务必使用 `long long`。
2. **DP 初始化**：题目中 $s_i$ 可能为负数，所以 `dp` 数组必须初始化为非常小的负数（如 `-1e18`），不能初始化为 0。
3. **内层循环顺序**：代码中内层循环 `j` 从 `i-1` 倒序遍历到 `0`。这是一个常数优化：利用 $x_i$ 递增的特性，一旦 $x_i - x_j > \text{max\_dist}$，就可以直接 `break`，这在 $g$ 较小时能显著减少计算量。

### 下一步建议

这个代码可以通过前 50% 的数据点（$n \le 500$）。如果你想挑战 **$n=5\times 10^5$ (100%分数)** 的满分做法，需要将内层的 $O(n)$ 查找优化为 $O(1)$。
