# 0. 这题到底在问什么

## Windy 数定义
一个数是 Windy 数，当且仅当：

1. 它没有前导零
2. 相邻两个数字之差至少为 `2`

比如：

- `135` 是 Windy 数，因为
  `|1-3|=2`，`|3-5|=2`
- `121` 不是，因为
  `|2-1|=1 < 2`
- `98` 不是，因为
  `|9-8|=1 < 2`

题目要求：

> 求区间 `[a,b]` 中有多少个 Windy 数。

---

# 1. 数位 DP 的标准套路

区间问题先转成前缀问题：

```cpp
答案 = solve(b) - solve(a - 1)
```

其中：

```cpp
solve(x) = [0, x] 中 Windy 数的个数
```

所以我们只需要先学会：

> 怎么求 `solve(x)`

---

# 2. 为什么可以用 DFS

我们来想：

假设 `x = 325`

那么所有不超过 `325` 的数，都可以通过“从高位到低位枚举每一位”来构造：

- 百位填 `0~3`
- 如果百位填了 `3`，十位只能填 `0~2`
- 如果前两位填成 `32`，个位只能填 `0~5`

这就是一棵“填数字”的树。

每个节点表示：

- 现在填到了第几位
- 前一位填了什么
- 当前是否受到上界限制
- 当前是否还处于前导零状态

所以自然就写成 DFS。

---

# 3. 先别急着写代码，先想状态

数位 DP 最重要的是：

> **每个状态要描述“后面的选择只和这些信息有关”**

这题，我们从高位往低位填。

那么 DFS 至少要知道：

---

## 状态 1：填到第几位了
记作 `pos`

比如总共 5 位数，现在处理第 5 位、第 4 位……直到第 1 位。

---

## 状态 2：前一位是什么
记作 `pre`

因为题目要求：

> 相邻位数字之差至少为 2

所以当前填 `d` 时，要和前一位 `pre` 比较：

```cpp
abs(d - pre) >= 2
```

---

## 状态 3：是否贴着上界
记作 `limit`

比如我们在算 `solve(325)`：

- 如果前面填得和 `325` 一样，那么这一位不能随便填，只能填到上界
- 如果前面已经比 `325` 小了，那么这一位可以随便填 `0~9`

所以：

- `limit = true`：这一位最大只能填到 `x` 对应这一位
- `limit = false`：这一位可以填 `0~9`

---

## 状态 4：前面是否全是前导零
记作 `lead`

这个非常关键。

比如你在枚举 `325` 以内的数时，实际上也会枚举：

- `007`
- `035`
- `123`

这些都是“补零后”的写法。

前导零不算真正的数字，所以：

- 如果当前还在前导零阶段，那么“上一位”其实还不存在
- 此时不能拿 `pre` 去做相邻差判断

---

# 4. 所以 DFS 应该长什么样

我们先写出函数外形：

```cpp
dfs(pos, pre, limit, lead)
```

表示：

> 当前正在处理第 `pos` 位，上一位是真实数字 `pre`，
> 当前是否受上界限制 `limit`，
> 当前是否仍处于前导零状态 `lead`，
> 返回满足条件的方案数。

---

# 5. 递归终点是什么

当所有位都填完了，也就是：

```cpp
pos == 0
```

说明构造出了一个完整的数。

这时应该返回多少？

如果这个数是合法构造出来的，就返回 `1`。

所以：

```cpp
if (pos == 0) return 1;
```

---

# 6. 当前位能填什么

如果当前受上界限制：

```cpp
up = 当前这一位最多只能填到 digits[pos]
```

否则：

```cpp
up = 9
```

于是我们枚举：

```cpp
for (int d = 0; d <= up; d++)
```

---

# 7. 枚举 `d` 时怎么判断合法

这是这题最核心的部分。

---

## 情况 A：还在前导零，并且当前继续填 0

如果：

```cpp
lead == true && d == 0
```

说明：

- 前面全是前导零
- 当前这一位也填 0
- 仍然没有真正开始这个数

比如：

- `000...0`
- `000...05`

这种情况下，当前位不算真实数字，
所以：

- 不需要检查和 `pre` 的差
- 下一层还是前导零状态

即：

```cpp
dfs(pos - 1, pre, 新limit, true)
```

注意这里 `pre` 其实无所谓，因为还没开始。

---

## 情况 B：当前位是第一个非零数字

如果前面一直是前导零，现在填了一个非零数字，比如 `d = 3`：

那它就是这个数的第一位。

第一位没有“前一位”可以比较，所以一定合法。

于是可以直接进入下一层：

```cpp
dfs(pos - 1, d, 新limit, false)
```

---

## 情况 C：已经开始填真实数字了

如果 `lead == false`，说明之前已经有真实数字了，
那么当前位 `d` 必须满足：

```cpp
abs(d - pre) >= 2
```

合法才继续：

```cpp
dfs(pos - 1, d, 新limit, false)
```

---

# 8. 先用自然语言写出完整转移

所以 `dfs(pos, pre, limit, lead)` 的逻辑就是：

1. 如果 `pos == 0`，返回 `1`
2. 枚举当前位 `d`
3. 如果还在前导零且 `d == 0`
   - 继续前导零
4. 否则如果之前还没开始，但现在 `d != 0`
   - 这是第一位，直接合法
5. 否则
   - 检查 `abs(d - pre) >= 2`

---

# 9. 为什么要记忆化

因为很多状态会重复。

比如：

- 你已经不受上界限制了
- 也已经开始填数字了
- 当前还剩 `pos` 位
- 上一位是 `pre`

那后面的方案数就是固定的。

所以可以记忆化：

```cpp
dp[pos][pre]
```

但注意：

> 只有当 `limit = false` 且 `lead = false` 时，才能直接记忆化这个状态

因为：

- `limit = true` 时，上界不同，答案不同
- `lead = true` 时，还没开始填，状态含义不完全一样，通常不直接存这个版本

---

# 10. 先把 `solve(x)` 的流程理清楚

假设要算 `solve(325)`：

## 第一步：把 `325` 拆成数位
通常拆成：

```cpp
digit[1] = 5
digit[2] = 2
digit[3] = 3
```

也就是**低位在前**。

然后调用：

```cpp
dfs(3, ?, true, true)
```

这里：

- `pos = 3`，表示从最高位开始填
- `limit = true`，因为一开始必须不能超过 `325`
- `lead = true`，因为一开始还没填任何真实数字

至于 `pre`，因为一开始没有前一位，随便传一个占位值即可，比如 `0`。

# 11. 现在我们正式写代码骨架

先写最完整、最容易理解的版本。

```cpp
#include <bits/stdc++.h>
using namespace std;

int a, b;
int digit[15];
int dp[15][15];

int dfs(int pos, int pre, bool limit, bool lead) {
    if (pos == 0) return 1;

    if (!limit && !lead && dp[pos][pre] != -1)
        return dp[pos][pre];

    int up = limit ? digit[pos] : 9;
    int res = 0;

    for (int d = 0; d <= up; d++) {
        if (lead && d == 0) {
            // 继续前导零
            res += dfs(pos - 1, pre, limit && (d == up), true);
        } else if (lead) {
            // 第一个非零位
            res += dfs(pos - 1, d, limit && (d == up), false);
        } else {
            // 已经有前一位了，要检查差值
            if (abs(d - pre) >= 2) {
                res += dfs(pos - 1, d, limit && (d == up), false);
            }
        }
    }

    if (!limit && !lead)
        dp[pos][pre] = res;

    return res;
}

int solve(int x) {
    if (x == 0) return 0;

    int len = 0;
    while (x) {
        digit[++len] = x % 10;
        x /= 10;
    }

    return dfs(len, 0, true, true);
}

int main() {
    memset(dp, -1, sizeof(dp));
    cin >> a >> b;
    cout << solve(b) - solve(a - 1) << '\n';
    return 0;
}
```

# 12. 这份代码你要逐行理解什么

下面我把每一部分拆开讲。

---

## `digit[]` 为什么低位在前

```cpp
while (x) {
    digit[++len] = x % 10;
    x /= 10;
}
```

比如 `325`：

- `digit[1] = 5`
- `digit[2] = 2`
- `digit[3] = 3`

这样我们调用 `dfs(len, ...)` 时，
`pos = 3` 对应最高位，正好方便从高位往低位递归。

---

## 为什么 `up = limit ? digit[pos] : 9`

如果当前仍受上界限制，那么这一位最多只能填到原数这一位。

否则已经比原数小了，可以随便填 `0~9`。

---

## 为什么 `limit && (d == up)` 能更新下一层的 `limit`

这个写法你一定要吃透。

如果当前这层本来就受限制：

- 且你填到了最大值 `up`
- 那说明前缀仍然和上界一样
- 下一层仍受限制

否则：

- 当前填得比上界小
- 后面就自由了

所以：

```cpp
new_limit = limit && (d == up)
```

---

# 13. 最关键的难点：`lead`

你必须理解：

> `lead` 不是说“这一位是不是 0”，
> 而是说“前面到现在为止，是否一直没有出现过非零数字”。

比如枚举 5 位数时：

- `00037`
  - 前三位都属于前导零
  - 到 `3` 才真正开始这个数

为什么要这样？

因为题目要求是“没有前导零的正常整数”，但在数位 DP 里为了统一长度，我们必须补零。
所以要用 `lead` 区分：

- 哪些 0 是前导零
- 哪些 0 是数中真实的一位

例如数 `101`：

- 中间的 `0` 不是前导零，必须参与相邻差判断

---

# 14. 为什么 `pos==0` 返回 1，不怕把 0 算进去吗？

这是一个很好的问题。

在这题里，`solve(0)` 我们手动返回了 `0`：

```cpp
if (x == 0) return 0;
```

所以不会出问题。

而对于 `solve(x)`（`x>0`），递归中如果一直走前导零到最后，相当于构造出了“0”。
但由于题目统计的是正整数范围，一般这里不会影响最终答案，或者也可特判。

这题通常像上面写就能过。

# 15. 我们来手推一个小例子

假设求 `solve(25)`。

数位是：

- `digit[1] = 5`
- `digit[2] = 2`

调用：

```cpp
dfs(2, 0, true, true)
```

---

## 第一层：十位
可填 `0~2`

### 填 `0`
还是前导零，进入：

```cpp
dfs(1, 0, false, true)
```

这表示统计 `0~9` 里合法的数。

### 填 `1`
这是第一个非零位，进入：

```cpp
dfs(1, 1, false, false)
```

接下来个位必须满足和 `1` 的差至少 2。

### 填 `2`
仍然受上界限制，进入：

```cpp
dfs(1, 2, true, false)
```

个位最多只能填 `5`，且要满足和 `2` 的差至少 2。

---

你会发现，这就是一棵树上的 DFS。

# 16. 现在你应该总结出“Windy 数状态设计”

这题最核心结论：

```cpp
dfs(pos, pre, limit, lead)
```

其中：

- `pos`：当前处理到第几位
- `pre`：上一位填的真实数字
- `limit`：是否贴上界
- `lead`：是否仍处于前导零阶段

---

# 17. 这题为什么只需要 `dp[pos][pre]`

因为当：

- `limit = false`
- `lead = false`

时，后面的结果只由：

- 还剩几位 `pos`
- 上一位是多少 `pre`

决定。

所以可以记：

```cpp
dp[pos][pre]
```

# 18. 给你一份更规范的最终代码

```cpp
#include <bits/stdc++.h>
using namespace std;

int a, b;
int digit[15];
int dp[15][10];

int dfs(int pos, int pre, bool limit, bool lead) {
    if (pos == 0) return 1;

    if (!limit && !lead && dp[pos][pre] != -1)
        return dp[pos][pre];

    int up = limit ? digit[pos] : 9;
    int res = 0;

    for (int d = 0; d <= up; d++) {
        if (lead && d == 0) {
            // 还没开始构造真正的数
            res += dfs(pos - 1, pre, limit && (d == up), true);
        } else {
            // 如果之前还在前导零，那么这是第一位，直接合法
            // 如果之前已经开始了，就要检查相邻位差值
            if (lead || abs(d - pre) >= 2) {
                res += dfs(pos - 1, d, limit && (d == up), false);
            }
        }
    }

    if (!limit && !lead)
        dp[pos][pre] = res;

    return res;
}

int solve(int x) {
    if (x <= 0) return 0;

    int len = 0;
    while (x) {
        digit[++len] = x % 10;
        x /= 10;
    }

    return dfs(len, 0, true, true);
}

int main() {
    memset(dp, -1, sizeof(dp));
    cin >> a >> b;
    cout << solve(b) - solve(a - 1) << '\n';
    return 0;
}
```
