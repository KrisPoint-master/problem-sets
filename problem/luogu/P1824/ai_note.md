# P1824 [USACO05FEB] 进击的奶牛 Aggressive Cows G 笔记

## 1. 题意概括

有 $n$ 间牛舍，位置分别为：

$$
x_1, x_2, \dots, x_n
$$

现在要把 $m$ 头牛放入这些牛舍中，要求：

- 每间牛舍最多放一头牛；
- 任意两头牛之间的距离尽可能远；
- 最终目标是最大化最近两头牛之间的距离。

也就是要求：

$$
\max \left( \min |x_i - x_j| \right)
$$

其中 $x_i, x_j$ 是被选中放牛的牛舍位置。

---

## 2. 解题思路

这道题是经典的 **二分答案** 问题。

我们不直接枚举每一种放牛方案，而是二分枚举一个距离 $d$，判断：

> 是否可以放下 $m$ 头牛，并且任意两头牛之间的距离都至少为 $d$。

如果可以，说明距离 $d$ 是可行的，可以尝试更大的距离。

如果不可以，说明距离 $d$ 太大了，需要尝试更小的距离。

---

## 3. 为什么可以二分

假设存在一个距离 $d$，使得可以安排 $m$ 头牛，并且相邻两头牛之间的距离至少为 $d$。

那么对于任何小于 $d$ 的距离，也一定可以满足。

也就是说，如果距离 $d$ 可行，那么：

$$
d' \le d
$$

时，$d'$ 也一定可行。

所以可行性具有单调性：

```text
true true true true false false false
```

也就是：

```text
T T T T F F F
```

我们的目标是找到最后一个 `true`。

也就是：

> 最大的可行距离。

---

## 4. 预处理：排序

题目中说明：

> 不保证 $x$ 数组单调递增。

所以在判断放牛之前，需要先对牛舍位置排序。

```cpp
sort(a + 1, a + 1 + n);
```

排序后，牛舍位置满足：

$$
a_1 \le a_2 \le \dots \le a_n
$$

这样才能从左到右贪心放牛。

---

## 5. 判断函数 `check(d)`

### 5.1 含义

```cpp
bool check(int d)
```

表示：

> 能否在所有牛舍中放下 $m$ 头牛，使得任意两头相邻牛之间的距离至少为 $d$。

如果可以，返回 `true`。

否则返回 `false`。

---

### 5.2 贪心策略

判断距离 $d$ 是否可行时，使用贪心：

1. 第一头牛放在最左边的牛舍；
2. 之后从左到右扫描；
3. 如果当前牛舍距离上一头牛的位置至少为 $d$，就在这里放一头牛；
4. 最后判断是否能放下至少 $m$ 头牛。

---

### 5.3 为什么贪心正确

对于固定的距离 $d$，想要尽可能多地放牛。

第一头牛放得越靠左，后面剩余空间越大，越有利于继续放牛。

因此每次都选择当前能放的最靠左牛舍，是最优的。

这个策略不会减少之后可放牛的数量。

所以如果贪心都放不下 $m$ 头牛，那么其他方案也放不下。

---

### 5.4 `check` 代码

```cpp
bool check(int d)
{
    int cnt = 1;
    int last = a[1];

    for (int i = 2; i <= n; i++)
    {
        if (a[i] - last >= d)
        {
            cnt++;
            last = a[i];
        }

        if (cnt >= m)
        {
            return true;
        }
    }

    return false;
}
```

---

### 5.5 变量解释

```cpp
int cnt = 1;
```

表示已经放了多少头牛。

一开始把第一头牛放在最左边的牛舍，所以 `cnt` 初始为 $1$。

```cpp
int last = a[1];
```

表示上一头牛所在的位置。

之后每次判断：

```cpp
if (a[i] - last >= d)
```

如果当前牛舍和上一头牛之间的距离至少为 $d$，说明可以放牛。

---

## 6. 二分答案 `bs_find`

### 6.1 二分区间

答案是一个距离。

最小可能距离为：

$$
0
$$

最大可能距离为：

$$
a_n - a_1
$$

因为两头牛之间的距离不可能超过最右边牛舍和最左边牛舍之间的距离。

所以初始二分区间为：

```cpp
bs_find(0, a[n] - a[1])
```

---

### 6.2 二分目标

我们要找的是最大的可行距离。

可行性形态是：

```text
T T T T F F F
```

所以使用“找最后一个 `true`”的二分模板。

---

### 6.3 推荐二分写法

```cpp
int bs_find(int l, int r)
{
    while (l < r)
    {
        int mid = (l + r + 1) >> 1;

        if (check(mid))
            l = mid;
        else
            r = mid - 1;
    }

    return l;
}
```

---

## 7. 二分边界解释

### 7.1 为什么循环条件是 `while (l < r)`

在这个写法中，区间 $[l, r]$ 表示当前答案可能存在的范围。

当：

$$
l < r
$$

说明区间里还有多个候选答案，需要继续二分。

当：

$$
l = r
$$

说明区间只剩一个值，这个值就是答案。

所以循环结束后返回：

```cpp
return l;
```

---

### 7.2 为什么不是 `while (l <= r)`

如果使用：

```cpp
while (l <= r)
```

再配合：

```cpp
if (check(mid))
    l = mid;
```

就可能出现死循环。

因为当 $l = r$ 时：

$$
mid = l = r
$$

如果 `check(mid)` 为真，执行：

```cpp
l = mid;
```

那么 $l$ 没有变化，循环无法结束。

所以这种无 `ans` 变量的写法通常使用：

```cpp
while (l < r)
```

---

### 7.3 为什么 `mid` 要上取中点

代码中使用：

```cpp
int mid = (l + r + 1) >> 1;
```

也就是：

$$
mid = \left\lfloor \frac{l + r + 1}{2} \right\rfloor
$$

这是上取中点。

因为我们在 `check(mid)` 成立时会写：

```cpp
l = mid;
```

如果使用普通中点：

```cpp
int mid = (l + r) >> 1;
```

当区间只剩两个数时，`mid` 会等于左边界 $l$。

如果此时 `check(mid)` 成立，再执行：

```cpp
l = mid;
```

区间不会缩小，可能死循环。

而上取中点可以保证在区间只剩两个数时，`mid` 取到右边界，从而保证区间继续缩小。

---

### 7.4 为什么 `check(mid)` 成立时写 `l = mid`

如果：

```cpp
check(mid) == true
```

说明距离 `mid` 可以实现。

因此答案至少是 `mid`。

由于我们要找最大可行距离，所以应该继续尝试更大的值。

此时答案一定在：

$$
[mid, r]
$$

所以写：

```cpp
l = mid;
```

不能直接写成：

```cpp
l = mid + 1;
```

因为 `mid + 1` 不一定可行，而 `mid` 本身是一个已经确认可行的答案，不能丢掉。

---

### 7.5 为什么 `check(mid)` 不成立时写 `r = mid - 1`

如果：

```cpp
check(mid) == false
```

说明距离 `mid` 太大，无法放下 $m$ 头牛。

由于距离越大越难满足，所以所有大于等于 `mid` 的距离也都不可行。

因此答案只能在：

$$
[l, mid - 1]
$$

所以写：

```cpp
r = mid - 1;
```

---

## 8. 完整代码

```cpp
#include <bits/stdc++.h>
using namespace std;

const int maxn = 1e5 + 5;

int n, m;
int a[maxn];

bool check(int d)
{
    int cnt = 1;
    int last = a[1];

    for (int i = 2; i <= n; i++)
    {
        if (a[i] - last >= d)
        {
            cnt++;
            last = a[i];
        }

        if (cnt >= m)
        {
            return true;
        }
    }

    return false;
}

int bs_find(int l, int r)
{
    while (l < r)
    {
        int mid = (l + r + 1) >> 1;

        if (check(mid))
            l = mid;
        else
            r = mid - 1;
    }

    return l;
}

int main()
{
    cin >> n >> m;

    for (int i = 1; i <= n; i++)
    {
        cin >> a[i];
    }

    sort(a + 1, a + 1 + n);

    cout << bs_find(0, a[n] - a[1]) << endl;

    return 0;
}
```

---

## 9. 数据范围与类型

题目给出：

$$
2 \le n \le 10^5
$$

$$
0 \le x_i \le 10^9
$$

因此牛舍位置最大为 $10^9$。

两个位置之差最大也是 $10^9$，在 `int` 范围内。

所以本题使用 `int` 可以通过。

不过如果想写得更稳，也可以使用 `long long` 存储位置和距离。

---

## 10. 时间复杂度

排序复杂度为：

$$
O(n \log n)
$$

每次 `check(d)` 需要从左到右扫描一遍数组，复杂度为：

$$
O(n)
$$

二分答案的范围最大约为 $10^9$，所以二分次数为：

$$
O(\log 10^9)
$$

总时间复杂度为：

$$
O(n \log n + n \log 10^9)
$$

通常也可以写作：

$$
O(n \log n + n \log V)
$$

其中 $V$ 表示坐标范围。

---

## 11. 空间复杂度

主要使用数组存储牛舍位置，空间复杂度为：

$$
O(n)
$$

## 12. 易错点总结

### 12.1 忘记排序

题目不保证输入的牛舍位置有序，所以必须先排序：

```cpp
sort(a + 1, a + 1 + n);
```

---

### 12.2 二分右边界写错

最大距离应该是：

```cpp
a[n] - a[1]
```

而不是：

```cpp
a[n]
```

因为距离是两个牛舍位置之差。

---

### 12.3 找最大可行值时要用上取中点

应该写：

```cpp
int mid = (l + r + 1) >> 1;
```

不要写成：

```cpp
int mid = (l + r) >> 1;
```

否则配合：

```cpp
l = mid;
```

可能死循环。

---

### 12.4 循环条件不要随便写成 `l <= r`

如果使用下面这种更新方式：

```cpp
if (check(mid))
    l = mid;
else
    r = mid - 1;
```

循环条件应该是：

```cpp
while (l < r)
```

不要写成：

```cpp
while (l <= r)
```

否则可能死循环。

---

### 12.5 `check(mid)` 成立时不能写 `l = mid + 1`

在当前模板中：

```cpp
if (check(mid))
    l = mid;
```

因为 `mid` 本身是可行答案，需要保留。

如果写成：

```cpp
l = mid + 1;
```

可能会跳过真正答案。

---

## 13. 模板总结

本题属于：

> 找最大的满足条件的值。

判断函数形态：

```text
T T T T F F F
```

对应模板：

```cpp
while (l < r)
{
    int mid = (l + r + 1) >> 1;

    if (check(mid))
        l = mid;
    else
        r = mid - 1;
}

return l;
```

记忆方式：

> 找最大，用上取中点；
> 可行就往右找，不可行就往左缩。
