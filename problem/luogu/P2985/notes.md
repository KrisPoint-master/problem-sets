# P2985 [USACO10FEB] Chocolate Eating S — 二分答案笔记

## 问题本质

在 $D$ 天内按顺序吃 $N$ 块巧克力，每晚幸福值向下取整减半。要求最大化 **所有天中最低的幸福值**（即每天吃完巧克力后的幸福值的最低值）。

## 二分答案

- **答案范围**：$[0, \sum H_i]$，下限 0，上限为所有巧克力幸福值之和。
- **判定函数 `check(d)`**：给定目标最低幸福值 $d$，判断是否存在一种吃巧克力的方案，使得每一天的幸福值都 $\ge d$。
- **单调性**：若 $d$ 可行，则所有 $< d$ 的值也一定可行（单调不降）。因此二分找出最大的可行 $d$。

## 判定过程

贪心模拟每一天：
1. 每天起床时幸福值为前一天的 $\lfloor \frac{fun}{2} \rfloor$。
2. 若当前幸福值 $\ge d$，标记当天达标，不强制吃巧克力。
3. 若当前幸福值 $< d$，按顺序吃巧克力，每吃一块 `fun += H_i`，直到 $\ge d$ 或巧克力吃完。
4. 若最终仍然 $< d$，当天不达标，`check(d)` 返回 `false`。
5. 所有 $D$ 天都达标才返回 `true`。

## 方案输出

二分找到最大可行 $d$ 后，重新运行 `check(d)` 并记录每块巧克力被吃掉的日子。未被吃掉的巧克力（因为某些天提前达标而不需要吃）统一分配到第 $D$ 天。

## 常见坑点

| 坑 | 说明 |
|---|---|
| **二分写法** | 找最大可行值，用 `if(check(m)) l=m+1 else r=m`，最后 `l-1` 是答案。 |
| **未吃巧克力的处理** | `check` 过程中可能因提前达标而跳过部分巧克力，输出时需要把所有未吃的分配到第 $D$ 天，**循环必须走到 `n` 而不是 `D`**（当 $N > D$ 时易漏）。 |
| **初始化** | 每日模拟前需要清空访问标记和日期记录数组。 |
| **顺序性** | 巧克力必须按输入顺序吃，不能乱序。 |
| **数据类型** | 幸福值累加可能达到 $5 \times 10^{10}$，需要用 `long long`。 |
