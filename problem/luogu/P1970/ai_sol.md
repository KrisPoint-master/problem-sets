这道题（NOIP 2013 花匠）本质上要求我们求出一个**最长的摆动子序列（Wiggle Subsequence）**。也就是说，子序列的元素必须是“大小交替”出现的（即：大-小-大-小…… 或 小-大-小-大……）。

下面为你详细解析 **$O(n^2)$ 的动态规划算法（类似 LIS 最长上升子序列）** 以及 **$O(n)$ 的最优解算法**。

---

## 1. 70% 数据算法：$O(n^2)$ 的动态规划 (类似 LIS)

类似最长上升子序列（LIS）的思想，我们可以通过动态规划来解决。由于摆动序列的状态有两种可能（最后一步是“上升”的，或者最后一步是“下降”的），我们需要用两个状态数组。

### 状态定义

* `up[i]`：表示以第 $i$ 株花结尾，且最后一株花比前一株花**高**的最长摆动序列长度。
* `down[i]`：表示以第 $i$ 株花结尾，且最后一株花比前一株花**低**的最长摆动序列长度。

### 状态转移方程

对于当前第 $i$ 株花，我们需要枚举它前面的某一株花 $j$（其中 $1 \le j < i$）：

1. 如果 $h[i] > h[j]$，说明可以从 $j$ 的“下降状态”转移过来，形成一个上升：

$$up[i] = \max(up[i], down[j] + 1)$$


2. 如果 $h[i] < h[j]$，说明可以从 $j$ 的“上升状态”转移过来，形成一个下降：

$$down[i] = \max(down[i], up[j] + 1)$$



### 初始状态与答案

* 初始化：所有的 `up[i]` 和 `down[i]` 都初始化为 1（单独一株花自成一个序列）。
* 最终答案：整个 `up` 和 `down` 数组中的最大值。

### 70分代码实现

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    if (!(cin >> n)) return 0;

    vector<int> h(n);
    for (int i = 0; i < n; ++i) {
        cin >> h[i];
    }

    vector<int> up(n, 1);
    vector<int> down(n, 1);
    int ans = 1;

    // O(n^2) 的双重循环
    for (int i = 1; i < n; ++i) {
        for (int j = 0; j < i; ++j) {
            if (h[i] > h[j]) {
                up[i] = max(up[i], down[j] + 1);
            } else if (h[i] < h[j]) {
                down[i] = max(down[i], up[j] + 1);
            }
        }
        ans = max({ans, up[i], down[i]});
    }

    cout << ans << "\n";
    return 0;
}

```

## 2. 100% 数据算法：$O(n)$ 的优化（线性贪心/DP）

题目中 $n \le 10^5$， $O(n^2)$ 的算法显然会超时。我们需要将时间复杂度降低到 $O(n)$。

### 思路一：线性动态规划优化

在 $O(n^2)$ 的算法中，我们为了求 `up[i]`，遍历了前面所有的 `down[j]`。但实际上，我们只需要知道**前面所有位置中 `down` 的最大值**。

我们可以重新定义状态，让状态本身就代表“前 $i$ 个数能组成的最长序列”：

* `up[i]`：前 $i$ 株花中，以**上升**趋势结尾的最长摆动序列长度。
* `down[i]`：前 $i$ 株花中，以**下降**趋势结尾的最长摆动序列长度。

由于我们只需要看它和**前一株花 $h[i-1]$** 的关系：

1. 如果 $h[i] > h[i-1]$：它可以接在之前的下降序列后面变成上升。
* `up[i] = down[i-1] + 1`
* `down[i] = down[i-1]`（没有下降，保持原样）


2. 如果 $h[i] < h[i-1]$：它可以接在之前的上升序列后面变成下降。
* `down[i] = up[i-1] + 1`
* `up[i] = up[i-1]`（没有上升，保持原样）


3. 如果 $h[i] == h[i-1]$：什么都不能改变。
* `up[i] = up[i-1]`
* `down[i] = down[i-1]`



因为每一次转移都只和 `i-1` 有关，我们甚至可以用两个变量 `up` 和 `down` 替代数组，空间复杂度直接优化到 $O(1)$。

### 思路二：贪心（寻找“极值点”）

如果我们把花的高度画成折线图，所谓的摆动序列，其实就是这条折线图上的**波峰**和**波谷**。

* 只要当前趋势在上升，我们就一直走到这个上升阶段的最高点（波峰）。
* 只要当前趋势在下降，我们就一直走到这个下降阶段的最低点（波谷）。
* 中间那些单调递增或递减的点，留在原地没有意义，直接移走不会影响结果。

因此，我们只需要遍历一遍数组，统计**方向发生改变的次数**（即波峰和波谷的数量）即可。

### 100分代码实现（线性 DP 滚动变量版）

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

int main() {
    // 优化输入输出流
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    if (!(cin >> n)) return 0;

    vector<int> h(n);
    for (int i = 0; i < n; ++i) {
        cin >> h[i];
    }

    if (n == 0) {
        cout << 0 << "\n";
        return 0;
    }

    // 初始状态，只有第 0 株花时，长度都为 1
    int up = 1;
    int down = 1;

    // 单重循环，时间复杂度 O(n)
    for (int i = 1; i < n; ++i) {
        if (h[i] > h[i-1]) {
            up = down + 1;
        } else if (h[i] < h[i-1]) {
            down = up + 1;
        }
    }

    // 最终答案为两种状态的最大值
    cout << max(up, down) << "\n";

    return 0;
}

```

## 总结

| 算法 | 时间复杂度 | 空间复杂度 | 得分 | 核心思想 |
| --- | --- | --- | --- | --- |
| **基础 DP (类LIS)** | $O(n^2)$ | $O(n)$ | 70分 | 枚举当前点之前的所有历史状态进行转移 |
| **优化 DP / 贪心** | $O(n)$ | $O(1)$ | 100分 | 状态只需从前一个位置转移 / 仅统计波峰与波谷 |
